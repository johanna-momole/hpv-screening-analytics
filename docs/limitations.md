# Limitations — HPV Screening Analytics Platform

## 1. Synthetic data

**All data in this project is fabricated.** Patient names, addresses, dates,
clinical results, and identifiers are generated by the Python Faker library
and custom random sampling routines. No real patient records, provider
records, or healthcare outcomes are used.

**Implication:** No descriptive statistics, model outputs, or visualizations
in this platform represent real populations. Do not use this project to
inform clinical decisions, policy, or public health reporting.

## 2. Embedded statistical signal

The data generation deliberately embeds associations between predictor
variables (insurance type, age, facility type, education level) and the
no-show outcome. This was done to make the model demonstration analytically
meaningful, not to replicate empirically validated risk estimates.

**Implication:** The model coefficients and risk scores reflect the synthetic
signal structure, not real-world evidence about no-show determinants.

## 3. Model training and evaluation on the same dataset

The no-show prediction model is trained and evaluated on the full synthetic
dataset. Cross-validation AUC provides a less biased estimate than training
AUC, but there is no true held-out test set.

**Implication:** Reported model performance metrics are illustrative. A
production deployment would require:
- A separate prospectively collected test set
- Temporal validation (train on earlier periods, test on later)
- Real-world outcome labeling and data quality checks

## 4. No clinical validation

Screening type assignments, result probability distributions, follow-up
rates, and time-to-follow-up distributions are approximated from general
knowledge of HPV screening workflows but are not calibrated to any specific
clinical study or population registry.

## 5. Missing data mechanisms

Missing values in the synthetic dataset are either absent (reference tables
are fully populated) or minimal (a small proportion of dates and notes are
left null). Real HPV screening databases exhibit complex missing data
patterns — particularly for race/ethnicity, insurance coverage, and
follow-up documentation — that are not modeled here.

## 6. Single-institution design

The schema models a network of hospitals and clinics but does not capture
real cross-facility data governance challenges (patient matching across
facilities, EHR interoperability, consent management, etc.).

## 7. No longitudinal patient tracking

Each patient's appointments are independent draws. The data does not model
realistic care trajectories (e.g., a patient progressing from abnormal Pap
→ colposcopy → biopsy → LEEP within the same episode of care).

## 8. Geographic scope

State assignments are drawn uniformly from 15 US states. The resulting
geographic distribution does not reflect real disparities in HPV screening
access across urban/rural or state-level insurance coverage contexts.
